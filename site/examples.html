Examples In Context

Examples that match the theory-first model

These examples are organized by conceptual goal, not just by app type. Use them to validate each stage of the Units mental model: structure, control flow, token-aware authoring, and advanced composition.

Read docs first Open examples directory

Concept Map

Use each example for a specific question

Question 1

What does a Units UI tree look like?

Run `learn-vite` first and follow template shape from top to bottom.

pnpm --filter units-learn-demo dev

Question 2

How do directives drive behavior?

Use `todo-vite` and `chat-vite` for loops, branches, and events.

pnpm --filter units-todo-demo dev
pnpm --filter units-chat-demo dev

Question 3

How does reusable composition scale?

Use `shadcn-gallery-vite` to inspect higher-level component composition.

pnpm --filter units-shadcn-gallery dev

Question 4

Where are fit boundaries in practice?

Try one UI edit in DSL and one in JSX to compare readability and token shape.

Review fit criteria

Core shape

learn-vite

Template structure and expression basics

  • Teaches: tags, text nodes, props, expression placement.
  • Does not teach: complex app-level architecture decisions.
  • Source: `examples/learn-vite/src/learn.ui`
Open source

Control flow

todo-vite

Conditions, lists, and events

  • Teaches: `#if`, `#for`, events, and predictable updates.
  • Does not teach: full-system IR or compiler workflows.
  • Source: `examples/todo-vite/src/todo.ui`
Open source

Nested UI

chat-vite

Transcript-like composition depth

  • Teaches: multi-level composition and branch-heavy display logic.
  • Does not teach: token benchmarking by itself.
  • Source: `examples/chat-vite/src/chat.ui`
Open source

Reusable layer

shadcn-gallery-vite

Higher-level component composition

  • Teaches: composing larger UI surfaces from reusable Units components.
  • Does not teach: parser internals or agent middleware.
  • Source: `examples/shadcn-gallery-vite/src/gallery.ui`
Open source
