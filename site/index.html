Units

UI DSL · Tooling · Agent pipeline

Small language. Broad surface area.

Units is a lightweight DSL for interactive UI, wrapped in a monorepo that now covers parsing, rendering, Vite integration, editor support, agent-facing compression tools, DOM extraction, IR compilation, and benchmark suites for both runtime and model workflows.

Units is a lightweight UI DSL with fast parsing, small syntax, and both React + custom renderer support.

What is in the repo now

Current monorepo surface

The GitHub Pages entry point now reflects the broader repo instead of only the original runtime and plugin packages.

Core runtime

Parse, print, render, incrementally update

  • @botfather/units: parser, printer, runtime, custom renderer, incremental sketch
  • React renderer plus host-agnostic createUnitsRenderer flow
  • Small grammar with raw JS expressions and directive support

Build and authoring

Vite, CLI, and editor tooling

  • @botfather/vite-plugin-units for .ui imports in Vite apps
  • @botfather/vite-plugin-units-tools for format, token, and highlight workflows
  • @botfather/units-tools plus the Units VS Code extension for .ui editing

Agent pipeline

Compression, DOM capture, IR, and compilation

  • @botfather/units-agent-middleware for agent-agnostic UI tree rewriting
  • @botfather/units-agent-plugin and @botfather/units-agent-service for compress-ui flows
  • @botfather/units-dom-snapshot, @botfather/units-ui-ir, @botfather/units-react-adapter, and @botfather/units-compiler

UI kit and reference apps

Examples that exercise the stack

  • @botfather/units-uikit-shadcn for a ShadCN-style component layer
  • todo-vite, chat-vite, shadcn-gallery-vite, learn-vite, and portfolio-vite
  • Bench fixtures and curated .ui/.jsx case pairs under bench/cases
