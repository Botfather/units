Level 1

Beginner — understand the shape of a Units template

Goal: read a .ui file, map it to a component tree, and render it once. Ignore optimization and advanced tooling for now.

You will learn

  • ◆How tags, text nodes, and props are written
  • ◆How a .ui file becomes an AST then rendered UI
  • ◆How to run the first demo locally

Do this first

pnpm install
pnpm --filter units-learn-demo dev
first-render.js
import { parseUnits, renderUnits } from "@botfather/units";

const src = "Card { text 'Hello Units' }";
const ast = parseUnits(src);
const ui  = renderUnits(ast, {});

Check yourself

Can you explain what parseUnits returns?
Can you explain where renderUnits gets its data context from?
Can you read examples/learn-vite/src/learn.ui without confusion?

Ready for Level 2 when all three feel intuitive.

Level 2

Intermediate — build interactive templates with directives

Now you express UI behavior: conditions, loops, and event-driven updates. Focus on readable state flow and predictability.

Key concepts

  • ◆#if, #else, #for for structure-level logic
  • ◆@expr for context-aware values
  • ◆Event handlers with !event notation

Hands-on apps

pnpm --filter units-todo-demo dev
pnpm --filter units-chat-demo dev
task-list.ui
#if (@items.length === 0) {
  Empty { text 'No tasks yet' }
} #else {
  #for (item in @items) {
    TaskCard title=@item.title !click { set(selected:=@item.id) }
  }
}

Check yourself

Can you add a filtered view (e.g. only "open" tasks) using directives instead of custom JS loops?
Can you modify todo/chat examples without copy-pasting from docs?
Can you debug expression errors at a glance?

Ready for Level 3 when all three hold in a real project.

Level 3

Advanced — integrate Units into your build flow

About team workflows: Vite integration, formatting/linting, and renderer choices. Optimizing developer experience and long-term maintainability.

Integration points

  • ◆Import .ui directly with @botfather/vite-plugin-units
  • ◆Format/lint with @botfather/units-tools
  • ◆Author with the VS Code extension for stable syntax support

Operational checks

pnpm --filter units-learn-demo build
pnpm --filter units-todo-demo  lint:ui
pnpm --filter units-chat-demo  preview

Check yourself

Can you explain where syntax validation, transformation, and rendering each happen?
Can your team run a repeatable .ui workflow from both CI and local dev?

Ready for Level 4 when both hold true in a production codebase.

Level 4

Expert — systems, pipelines, and baselines

Work across representation layers: DOM snapshots, UI IR, compilation targets, and agent middleware. Evaluate tradeoffs with benchmark suites.

Expert packages

  • @botfather/units-agent-middleware
  • @botfather/units-agent-plugin
  • @botfather/units-agent-service
  • @botfather/units-dom-snapshot
  • @botfather/units-ui-ir
  • @botfather/units-compiler

Benchmark workflow

pnpm bench:dsl
pnpm bench:llm
pnpm bench:react-vs-dsl
pnpm bench:ui-ps
node --test

Check yourself

□Can you justify a pipeline decision using measurable outcomes — tokens, latency, or maintainability?
□Can you design a custom renderer host from scratch?
□Can you write agent middleware that safely rewrites UI trees?

You are operating at expert level when benchmarks inform architecture decisions.

Reference

Glossary + next actions

Core terms

Template
A .ui source file with structure and expressions.
AST
Parsed tree form used for transforms and rendering.
IR
Neutral representation for cross-tool operations.
Middleware
Layer that rewrites or normalizes UI structures before rendering.
